# MN4 跨脑图答案匹配

为每个题目脑图绑定一个答案脑图。选中题目卡片后，通过插件按钮按标准化标题精确匹配答案卡片，并展示评论、摘录或子卡片内容。

## 安装

直接双击 `dist/mn4-answer-matcher-v2.1.0-beta.1.mnaddon`，由 MarginNote 4 导入。

插件每 12 小时自动检查一次 [GitHub Releases](https://github.com/sinner7620/mn4_answer_matcher/releases)，发现正式版或测试版后可直接下载并交给 MarginNote 覆盖安装；也可从插件菜单选择“检查插件更新”。覆盖升级会保留原有脑图绑定记录，并在安装前额外写入恢复备份。

## 使用

1. 打开题目脑图，点击工具栏中的“答案匹配”图标，打开 MN Rails 工作台。
2. 首次使用可在“维护”页打开经典功能菜单，选择“绑定/更换答案脑图”。
3. 选中一张题目卡片，在卡片右侧出现的蓝色悬浮按钮中点击“查找答案”；右侧空间不足时按钮会自动切换到左侧。
4. 若同标题答案不止一张，先从列表中选择；答案会以可滚动的完整卡片悬浮层显示。拖动顶部区域可将窗口自由移出屏幕边界，拖动右下角 `↘` 可调整大小。若窗口被完全移出，再次查找答案会自动恢复到屏幕中央。

顶部插件图标用于打开可拖动、可缩放并记忆位置的 React 工作台；日常查询仍可直接使用卡片旁的“查找答案”按钮。

## MN Rails 工作台

从 v2.1.0 起，插件迁移到 MN Rails 0.5.8 Web 模板架构。原生 JavaScriptCore 层继续负责 MarginNote 数据库、卡片操作、存储和 OTA；React WebView 只负责答案核对、错题分类、筛选、到期复习和维护界面，两层通过 MN Rails bridge 通信。

工作台不会使用 WebView `localStorage` 保存绑定或错题数据，升级时仍沿用原 `addonid` 和原生存储键，因此覆盖安装不会清空答案绑定、答案索引和错题记录。

## 错题系统

插件采用图片所示的 0–5 级错题分类。选中卡片后点击侧边的“错题分级”，选择等级即可把完整题目卡复制到统一的总错题脑图。首次使用前，请先在 MarginNote 新建一个空白脑图，再从插件菜单选择“绑定/更换总错题脑图”。

每张错题会保存首次记录时间、来源脑图、章节路径、原题标题、当前等级、下次复习时间，并建立“原题 ↔ 错题”双向链接。侧边的“原题 / 错题”按钮可以在两张卡之间跨脑图跳转；摘录完成后也可立即核对答案或打开错题。总错题脑图查答案时，插件根据来源卡片动态使用其原题脑图绑定的答案脑图，因此答案始终实时读取，不复制旧答案。

菜单“错题分类目录”按“来源脑图 → 章节”分组，并用自然题号稳定排序。升级已有记录后，执行一次“修复并整理错题记录”，即可补齐分类标签和双向链接；该操作不会删除或重新复制卡片。

不同等级使用不同复习间隔。插件在 MarginNote 启动、回到前台时检查到期错题，并在应用保持打开时每 30 分钟检查一次；提醒最多每 6 小时出现一次。菜单“错题统计与到期复习”可以查看各等级数量、到期数量并记录复习后的新等级。

答案卡片会显示标题、摘录、文字/HTML/图片评论、`LinkNote.q_htext` 合并文字、`LinkNote.q_hpic` 合并图片、旧格式递归合并内容和直接子卡片。循环合并引用会自动去重。图片从 MarginNote 数据库读取并以内嵌方式显示。带有 `标准答案` 标签的匹配卡片排在前面。

答案脑图中若存在已经失效的合并卡片引用，插件会忽略该引用并继续建立索引；单张异常卡片不会导致整张脑图绑定失败。

答案索引只在绑定答案脑图和手动选择“刷新答案索引”时建立，并会持久保存；正常打开脑图和查找答案不会重新扫描整张答案脑图。首次从旧版本升级后若提示索引尚未建立，手动刷新一次即可。


## 开发

```powershell
pnpm install
pnpm check
pnpm test
pnpm build
```
